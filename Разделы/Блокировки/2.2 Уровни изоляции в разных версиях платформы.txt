1. При записи, удалении, изменении данные блокируются всегда до конца транзакции

2. Автоматический режим блокировок
	
	READ UNCOMMITED - вне транзакции, данные не блокируются при чтении, есть грязное чтение
	
	REPEATABLE READ - в транзакции для ссылочных типов (документ, справочник, ПВХ и т.д.)
					  Данные при чтении блокируются до конца транзакции
	
	SERIALIZABLE -  в транзакции для регистров, наборов записей последовательностей
					Данные при чтении блокируются до конца транзакции, но блокируется диапазон прочитанных данных
					При чтении одной записи блокируется дополнительно еще и следующая по ключу индекса запись
	
3. Управляемый режим 8.2 и ранее	

	READ UNCOMMITED - вне транзакции, данные не блокируются при чтении, есть грязное чтение
	READ COMMITED 	- в транзакции для всех, данные при чтении блокируются на момент чтения
					после того, как запись прочитана, блокировка снимается

4. Управляемый режим 8.3 	

	READ COMMITED (SNAPSHOT) - данные не блокируются при чтении, 
							  читаются только подтвержденные версии записей
	
5. Управляемый режим + RCSI в 8.2 и ранее	

	READ UNCOMMITED 		- вне транзакции, данные не блокируются при чтении, есть грязное чтение 
	READ COMMITED (SNAPSHOT)- в транзакции для всех, данные не блокируются при чтении, 
							 читаются только подтвержденные версии записей
